SYNTAX RULES
& SEMICOLONS

JavaScript has strict syntax rules. Breaking them causes errors before your code even runs. This lesson covers the most common mistakes beginners make.

// WHY SEMICOLONS MATTER

Every JavaScript statement should end with a semicolon ;. JavaScript has Automatic Semicolon Insertion that guesses where statements end — but it does not always get it right. Always add your own.

EXAMPLE — CORRECT
let score = 100;
let name = "Player One";
console.log(score);
// VARIABLE ASSIGNMENT

When you declare a variable with let or const, use = to assign a value. Forgetting = is one of the most common beginner errors.

EXAMPLE — ERROR
let score 100; // ← missing =
const name = "Brian"; // ← correct
